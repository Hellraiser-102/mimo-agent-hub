# MiMo Agent Hub — 架构文档

## 系统总览

MiMo Agent Hub 是一个基于 OpenClaw 框架的多模型智能路由 Agent 平台，以小米 MiMo V2.5 Pro 为核心推理引擎。

## 核心架构

```
┌─────────────────────────────────────────────────────┐
│                    用户请求层                         │
│         (Control UI / Telegram / CLI)               │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│                 OpenClaw Gateway                     │
│            认证 · 路由 · 会话管理                     │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│               智能路由分发层                          │
│                                                     │
│  任务分类器                                          │
│    ├─ 重度推理 → MiMo V2.5 Pro                     │
│    ├─ 日常对话 → MiMo V2.5                         │
│    └─ 快速响应 → DeepSeek V4 Flash                 │
│                                                     │
│  故障处理                                            │
│    ├─ 超时控制 (AbortController, 30s)               │
│    ├─ 指数退避重试 (1s → 2s → 4s)                  │
│    └─ 自动降级 (429/5xx → 下一档模型)               │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│                  插件生态层                           │
│                                                     │
│  Tavily        — 联网搜索                           │
│  Active Memory — 长期记忆管理                        │
│  Memory Wiki   — 知识库构建                          │
│  Doc Extract   — 文档内容提取                        │
│  Web Reader    — 网页内容解析                        │
│  LLM Task      — 子任务编排                          │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│                  输出与执行层                         │
│                                                     │
│  消息回复 · 文件操作 · 定时任务 · 跨会话通信         │
└─────────────────────────────────────────────────────┘
```

## 路由决策逻辑

### 三档分类

| 档位 | 模型 | 触发条件 | 设计理由 |
|------|------|----------|----------|
| 重度推理 | MiMo V2.5 Pro | 长文本(>2000字)、代码相关关键词、分析类关键词 | Pro 模型推理能力强，适合复杂任务 |
| 日常对话 | MiMo V2.5 | 默认（不匹配重度和快速） | 标准版性价比高，适合大多数场景 |
| 快速响应 | DeepSeek V4 Flash | 短文本(<100字)、简单指令关键词 | Flash 模型速度快、成本低 |

### 关键词匹配规则

**重度推理特征：**
- 代码相关：代码、函数、实现、编写、重构、调试、review
- 分析类：分析、对比、评估、设计、架构、方案
- 长文本：包含代码块(>50字符)、长段落解释/总结

**快速响应特征：**
- 短文本(<100字)
- 简单问候：hi、hello、你好、ok
- 简单指令：翻译、格式化、转换、列出

## 故障降级流程

```
请求 MiMo Pro
  ├─ 成功 → 返回结果
  └─ 失败
      ├─ 超时 → 重试(1s) → 重试(2s) → 降级到 MiMo 标准版
      ├─ 429  → 立即降级到 MiMo 标准版
      └─ 5xx  → 立即降级到 MiMo 标准版

请求 MiMo 标准版
  ├─ 成功 → 返回结果
  └─ 失败 → 降级到 DeepSeek Flash

请求 DeepSeek Flash
  ├─ 成功 → 返回结果
  └─ 失败 → 抛出错误（所有模型不可用）
```

## 模型配置

| 模型 | 用途 | 超时 | 最大输出 |
|------|------|------|----------|
| MiMo V2.5 Pro | 重度推理主力 | 30s | 4096 tokens |
| MiMo V2.5 | 日常对话 | 20s | 2048 tokens |
| DeepSeek V4 Flash | 快速响应 | 15s | 2048 tokens |
| DeepSeek V4 Pro | 备用质量模型 | 30s | 4096 tokens |

  └─ Docker Desktop
      └─ openclaw-2026 容器
          ├─ OpenClaw Gateway (:18789 → :18999)
          ├─ Node.js Runtime
          ├─ 插件系统
          └─ 工作空间 (/home/node/.openclaw/workspace)

外部 API:
  ├─ MiMo API (token-plan-cn.xiaomimimo.com)
  ├─ DeepSeek API (api.deepseek.com)
